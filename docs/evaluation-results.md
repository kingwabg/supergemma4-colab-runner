# Gemma 4 12B 품질 평가 결과

## 남은 실패와 해석

- R012 중앙값과 R014 KST→UTC 변환은 모델 단독 계산의 실행 간 변동입니다.
- F005는 정확히 네 줄을 요구했지만 각 줄에 설명을 덧붙인 형식 실패입니다.
- 이 세 유형을 결정적으로 처리하는 중앙값·KST 계산기와 정확한 줄 수 출력 계약은 v4 코드에 추가했고 단위 테스트를 통과했습니다.
- 현재 Colab 세션의 v4 실행은 Python이 이전 `output_contract` 하위 모듈을 캐시한 상태여서 마지막 세 보강을 사용하지 않았습니다. 새 런타임에서 노트북을 처음부터 실행하면 최신 도구가 로드됩니다.

## 정직한 결론

이 결과는 작은 고정 업무 평가에서 에이전트 스택이 목표를 달성했다는 증거입니다. 모델 자체 가중치의 지능이 96%가 된 것은 아닙니다. 장문 설계, 새로운 도메인 지식, 복잡한 디버깅과 도구 사용은 별도 실제 업무 평가가 필요하며, 외부 전문가 API 폴백은 여전히 기본 OFF입니다.
