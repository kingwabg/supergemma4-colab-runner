# Feature Specification: Spec-driven Quality Agent

## 목표

무료 Colab T4에서 실행되는 로컬 GGUF 모델이 문서 근거, 자기 검증, 자동 재시도, 선택적 강한 API 폴백을 사용하고 실제 업무형 75문항으로 품질을 측정하게 한다.

## 사용자 시나리오

### P1 근거 기반 업무 질문

사용자는 TXT, Markdown, 코드, JSON, CSV, 텍스트 PDF를 넣고 출처가 표시된 답변을 받는다.

### P1 검증과 자동 수정

사용자는 초안이 검증에 실패했을 때 문제를 반영한 두 번째 답변을 자동으로 받는다.

### P2 비용 제한 폴백

사용자가 외부 API 설정을 명시적으로 켠 경우에만 로컬 최종 실패를 외부 전문가 모델로 보낸다.

### P2 사양 기반 작업 설계

사용자는 자연어 요청에서 헌법, 요구사항, 계획, 작업, 일관성 분석 파일을 생성한다.

### P1 반복 가능한 평가

사용자는 75개의 동일한 평가 문항으로 모델·설정 변경 전후 점수와 실패 사례를 비교한다.

## 수용 기준

- RAG 답변은 `[근거 N]`을 포함하고 검색 결과가 없으면 모른다고 답한다.
- 검증 실패 시 최소 한 번 자동 재시도하고 최종 검증 결과를 남긴다.
- 외부 API는 기본 OFF이고 로컬 최종 실패일 때만 호출된다.
- 75문항 평가는 중간 결과를 저장하고 같은 모델·평가셋에서 재개된다.
- Spec 워크플로는 5개 Markdown 산출물과 JSON manifest를 저장한다.
- OpenAI 호환 API는 direct, verified, hybrid 품질 모드를 제공한다.

## 범위 제외

- Colab 런타임의 상시 서비스 운영
- 생성된 셸 명령 또는 코드를 무승인 자동 실행
- 평가 점수를 범용 지능 또는 Codex 대비 절대 백분율로 해석
